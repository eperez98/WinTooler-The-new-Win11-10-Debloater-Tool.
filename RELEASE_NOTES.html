Full Change Log
NEW

ISO Creator — complete pipeline

Replaces the old download-based approach. Accepts an official Microsoft ISO, mounts it with Mount-DiskImage, strips read-only attributes from the copy, patches the WIM(s) with DISM, injects optional files, then rebuilds with oscdimg.

NEW

Bloatware removal via DISM /Get-ProvisionedAppxPackages

Queries real full package names from the mounted WIM before removing. Matches 24 short-name prefixes against the live list — works correctly across Win11 22H2, 23H2, 24H2 and 25H2 regardless of version suffix changes.

NEW

Network driver injection

Checkbox + folder picker. Runs DISM /Add-Driver /Driver:"path" /Recurse against the mounted install.wim. Shown/hidden dynamically when the checkbox is toggled.

NEW

App Package embedding in ISO

Popup window with full app catalog, search, category filter, and persistent selection. Uses System.Text.StringBuilder + File.WriteAllText for the batch script to avoid PS5.1 array type issues. XPath-based XML patching for autounattend.xml integration.

NEW

Delete All Restore Points (Repair page)

Runs vssadmin delete shadows /for=C: /all /quiet then sweeps Win32_ShadowCopy via WMI. Cycles Disable/Enable to flush the restore point registry list. Confirmation dialog before destructive action.

IMP

Full dark mode coverage

Apply-Theme now repaints Startup page rows and bottom bar, DNS cards and inputs, Backup cards and inputs, ISO status log panel, divider and output box, and the ISO app picker popup. SearchBox style no longer hardcodes white background.

IMP

ISO pipeline runs in background job

The entire ISO creation process runs in Start-Job with a DispatcherTimer draining output every 600ms. The GUI stays fully responsive during multi-gigabyte WIM operations. Progress prefixes: PROGRESS:, DONE:, ERROR:, LOGINFO:.

IMP

Read-only attribute strip after ISO mount

Immediately after Copy-Item from the mounted ISO drive, all file attributes are stripped of the ReadOnly flag. A second explicit check on install.wim before DISM mount prevents error 0xC1510111.

FIX

PS5.1 inline-if elimination

All .Property = if (...) and $var = if (...) inside ForEach-Object pipelines and stored scriptblocks converted to explicit if { } else { } blocks. Eliminates "The term 'if' is not recognized" crashes across Startup Manager, App Manager, Tweaks, DNS, and ISO pages.

FIX

App picker category filter scope

WPF mouse event handlers run in script scope, not function scope. All shared picker state moved to $script:-scoped variables (ISOPopCurCat, ISOPopRows, ISOPopLocalSel, etc.) and helper functions registered as script: functions. Category pills now filter correctly.

FIX

Startup Manager crash on Refresh List

All five inline-if assignments inside the ForEach-Object pipeline in $script:StartupBuildList converted to explicit if/else blocks.

FIX

Write-WTLog not found in background job

A self-installing Write-WTLog stub was added to each ISO module file. When the function is not already defined (background job context), it outputs LOGINFO:[prefix] message to the job stream which the timer handler strips and displays cleanly.

FIX

Receive-Job duplication

Removed -Keep from the timer's Receive-Job call. Previously every tick re-read all output from the beginning of the job, causing each log line to appear once per 600ms tick.

FIX

Object[] to UInt32 conversion error in winget embed

Replaced array-based string split and Out-File with StringBuilder + [System.IO.File]::WriteAllText. XML patching replaced with SelectSingleNode + XmlNamespaceManager + XPath to avoid member enumeration on array results.

IMP

CLI banner engine renamed

Engine : WinUtil Modified Framework → Engine : WinTooler Native Engine v0.7

IMP

ISO page layout redesigned

Step 1 requires a source ISO (no download fallback). Language dropdown removed from visible UI. Steps renumbered. ISO required validation with clear error message. "How it works" and requirements cards updated to reflect the new pipeline.

Known Limitations

ISO Creator requires DISM (built-in) or Windows ADK for oscdimg. ADK auto-install may require a system restart before the first ISO build.

App embedding in ISO requires winget to be available on the target system after Windows installs. The generated Install-Apps.bat checks for winget at runtime.

Chocolatey integration requires a separate manual install. winget is the primary package manager for App Manager.

Startup Manager task disabling requires the Windows Task Scheduler service to be in a running state.

Segoe MDL2 Assets / Segoe Fluent Icons may not render on Windows 10 builds before 19041. Unicode ASCII fallback characters are used in those cases.
