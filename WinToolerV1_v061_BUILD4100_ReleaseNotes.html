Critical Bug Fixes — App Manager
Fix
Apps never downloaded — root cause found
Start-Process -NoNewWindow fails silently inside Start-Job. Background jobs have no console window — the flag can't attach to anything and the process exits immediately. Replaced with the & call operator which always blocks correctly in any execution context.
Fix
Install job returned 0 results despite running
Hashtable values serialized across job boundaries become Deserialized.System.Collections.Hashtable — dot-notation .Success always returns $null. Jobs now emit [PSCustomObject] and comparison uses explicit -eq $true.
Fix
Only first app installed when multiple selected
Start-Job -ArgumentList ($array, $str) unrolls the array — each element maps to a separate param instead of the whole list. Now serialized as a compact JSON string and deserialized with ConvertFrom-Json inside the job.
Fix
Winget path resolution fails inside elevated jobs
Test-Path on winget's WindowsApps symlink can return $false in an elevated background job. Now falls back to Get-Command winget automatically if the path test fails.
Fix
Microsoft Store apps always failed to install
IDs like 9PF4KZ2VN4W9 (TranslucentTB), 9NNCB5BS59PH (Tidal), 9P07XNM5CHP0 (Ambie) are Store product IDs — they need --source msstore not --source winget. Now auto-detected by regex ^[0-9A-Z]{10,16}$.
Additional Fixes
Fix
3 incorrect winget package IDs corrected
starship → Starship.Starship  ·  netbird → Netbird.Netbird  ·  ForceAutoHDR.7gxycn08 → 7gxycn08.ForceAutoHDR (publisher and package were reversed)
Fix
UTF-8 parse error at GUI startup
Box-drawing Unicode chars in PS comments caused PowerShell 5.1 to read the file as Windows-1252. Byte 0x94 in Windows-1252 is " (right double quote), corrupting the parser. File re-saved as UTF-8 with BOM (EF BB BF); all non-ASCII removed from PS code.
Fix
DispatcherTimer crash on install/uninstall
Removed .GetNewClosure() from all DispatcherTimer.Add_Tick handlers. Job IDs are now stored as $script:AMInstallJobId / $script:AMUninstJobId for reliable cross-scope access.
Fix
Category click crash in App Manager
Eliminated .GetNewClosure() from all WPF event handlers. Per-element data (category names, app IDs) now stored in the element's Tag property and read via $this.Tag inside handlers.
New in Build 4100
New
376-app catalog
Expanded from 111 to 376 curated apps across 9 categories, sourced from ChrisTitusTech/winutil. Includes full Utilities, Multimedia, Development, Document, Microsoft Tools, Communications, Games, Pro Tools, and Browsers libraries.
New
Install diagnostic log
Every install session writes a timestamped log to %TEMP%\WinTooler_install_job.log — records the resolved winget path, every app attempted, arguments passed, and exit codes returned.
New
Chocolatey fallback
If winget fails for an app that has a Chocolatey ID in the catalog, WinTooler automatically retries via choco install before reporting failure.
